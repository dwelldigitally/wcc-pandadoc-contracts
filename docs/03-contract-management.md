# PandaDoc Contract Management

## Contract Lifecycle Stages

PandaDoc manages contracts through 6 stages:
1. **Drafting** — Create from templates with variables
2. **Negotiation** — Redlining and suggest-edits workflow
3. **Approval** — Internal approval routing before sending
4. **Signing** — eSignature collection
5. **Storage** — Centralized, searchable contract repository
6. **Renewal** — Automated renewal date notifications

## Renewals

- Set renewal date notifications to get alerted when contracts approach end dates
- Use for triggering re-enrollment workflows each semester/year
- Create renewal templates that reference original agreements

## Amendments

- Track revisions after contract completion
- Every change is logged with version history
- Both parties can see amendment history

## Versioning

- Each document maintains full version history of all pre-send edits
- Critical for compliance — proves which version a student signed
- Versions visible to all parties

## Contract Repository

- Centralized, searchable storage for all executed contracts
- Automated version tracking
- Instant access for audits and reviews

## Enrollment Strategy

1. Create a **master enrollment agreement template**
2. Each term, clone or bulk-send it
3. For mid-year term changes, create **amendment documents** referencing the original
4. Both original and amendments stored in repository
